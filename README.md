# 📱 Система

## 🚀 1. Быстрый старт
1. Создай пустую БД в pgAdmin (название по умолчанию `banketam_db`).
2. В терминале 1: `cd backend` -> `npm install` -> `npm run dev`.
3. В терминале 2: `cd frontend` -> `npm install` -> `npm run dev`.
*(Учетная запись админа Admin26 / Demo20 создается автоматически).*

## 🛠 2. случай провальный

Если на экзамене поменялась тема (например, вместо «Банкетов» нужно сделать «Бронирование отелей» или «Аренду авто»), действуй строго по этому алгоритму.

### ШАГ 1: База Данных (Backend)
Тебе нужен только **ОДИН** файл: `backend/src/models/models.js`.
1. Модель `User` не трогай, она нужна всегда.
2. Найди модель `Booking` и переименуй её (например, в `Order` или `Rent`).
3. Замени поля внутри. Например:
   *БЫЛО:* `room_type: { type: DataTypes.STRING }`
   *СТАЛО:* `car_model: { type: DataTypes.STRING }`
4. Сохрани файл. **В pgAdmin ничего делать не надо!** Благодаря параметру `alter: true` в `index.js`, Sequelize сам создаст новые колонки и таблицы.

### ШАГ 2: Контроллеры и Роуты (Backend)
1. Открой `backend/src/controllers/bookingController.js`.
2. В функции `create` поменяй получение данных из тела запроса на новые:
   *БЫЛО:* `const { room_type, event_date, payment_method } = req.body;`
   *СТАЛО:* `const { car_model, rent_date, payment_method } = req.body;`
3. Убедись, что при создании записи (`await Model.create({...})`) ты передаешь именно эти новые переменные.
4. Открой `backend/src/routes/index.js` и просто поменяй путь, если нужно (например, `router.use('/rent', rentRouter);`).

### ШАГ 3: Форма создания (Frontend)
Открой `frontend/src/pages/CreateBooking.jsx`.
1. Найди `useState` и измени ключи на новые (те самые, что ты прописал в бэкенде):
   `const [formData, setFormData] = useState({ car_model: 'BMW', rent_date: '', payment_method: 'Наличные' });`
2. Спустись к HTML-тегам `<input>` и `<select>`. У каждого из них есть атрибут `name`. **Атрибут `name` должен строго совпадать с ключами из `useState`!**
   *БЫЛО:* `<select name="room_type" ...>`
   *СТАЛО:* `<select name="car_model" ...>`
3. В функции `handleSubmit` проверь URL запроса (`await $host.post('/твой-новый-путь', formData);`).

### ШАГ 4: Вывод списков (Frontend)
Открой `Dashboard.jsx` (для юзера) и `Admin.jsx` (для админа).
1. Найди место, где происходит рендер (`bookings.map(booking => ...)`).
2. Поменяй вывод текста:
   *БЫЛО:* `{booking.room_type}`
   *СТАЛО:* `{booking.car_model}`
3. Готово! Приложение полностью переделано под новую тему.
